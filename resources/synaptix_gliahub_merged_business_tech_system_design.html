synaptix / gliahub için birleşik business, ürün ve teknik sistem tasarımı

bu tek dosyalık html; ticari due diligence raporu, teknik stack/scope raporu ve son mimari kararlarımızı birleştirir. ana yön: eeg/meg-first başlayan, marketplace destekli, graph tabanlı low-code/no-code workflow repository platformu.

tek cümle

datasetleri, workflow graph’larını ve execution run’larını immutable snapshot olarak versiyonlayan; go control plane ile izinleri, marketplace entitlement’larını ve run state’i yöneten; izole python container’larıyla domain plugin’lerini çalıştırıp artifact, report ve provenance üreten bilimsel workflow repository platformu.

karar özeti

raporların ana önerilerini koruyoruz ama mvp implementation tarafında daha pragmatik bir çizgi seçiyoruz: full k8s/argo başlangıcı yerine docker tabanlı custom go runner ile başla; aynı execution contract’ı ileride argo/kubernetes backend’e taşı.

versioning

git wrap değil; git-like immutable snapshot + manifest + provenance

runtime

go control plane + python execution containers

runner

docker sdk first; containerd/gvisor/k8s/argo later

plugin spec

canonical json schema + python sdk; yaml optional; xml only adapters

business

open-source core + paid lab workspace + marketplace + support

bölüm 6

versioning: git wrap değil, git-like product model

git kod için mükemmel ama 2 tb fmri/eeg raw data ve binary artifacts için core primitive olmamalı. ürün kaggle notebook versioning gibi “save version / restore / fork / run history / compare” deneyimi vermeli, fakat arka tarafta manifest + immutable snapshot + content hash çalışmalı.

workflow_version

immutable graph snapshot; parent version, author, message, graph hash, locked dependencies, node versions.

dataset_snapshot

immutable manifest; object keys, paths, sizes, checksums, bids/nwb metadata, validation summary.

execution_run

workflow_version + dataset_snapshot + params + runtime image digest + environment = reproducible run.

bölüm 7

graph workflow modeli

ürün knime benzeri graph tabanlı low-code/no-code workflow editor olacak. fakat her node sadece ui kutusu değil; typed, versioned ve executable contract.

node contract alanıaçıklama
typeör: eeg.mne.filter, fmri.nilearn.glm, tabular.duckdb.query.
portsinput/output type contract: mne.Raw, bids.Dataset, nifti.Image, parquet.Table, video.File.
params_schemajson schema ile form generation ve validation.
runtimeplugin package, version, image digest, python version, resource hints.
permissionsnetwork, secrets, filesystem scope, gpu, external api permissions.
provenancenode started/completed/failed, input artifacts, output artifacts, environment digest.

  "id": "node_filter_1",
  "type": "eeg.mne.filter",
  "plugin": "synaptix/eeg-core",
  "plugin_version": "1.2.0",
  "inputs": { "raw": "artifact://node_load_1/raw" },
  "params": { "l_freq": 1.0, "h_freq": 40.0, "method": "fir" },

canonical plugin manifest: json

  "package": "synaptix/eeg-core",
  "version": "1.2.0",
  "domain": ["eeg", "meg"],
  "runtime": {
    "image": "registry.synaptix.io/eeg-core@sha256:abc",
    "python": "3.11"
  },
  "nodes": [{
    "id": "eeg.mne.filter",
    "inputs": { "raw": { "type": "mne.Raw" } },
    "outputs": { "filtered": { "type": "mne.Raw" } },
    "params_schema": { "type": "object", "properties": { "l_freq": { "type": "number" } } },
    "permissions": { "network": false, "filesystem": "workspace-scoped", "secrets": [] }
  }]
}

json vs xml vs sdk kararı

canonical: json schema

ui form generation, validation, node contract ve marketplace metadata için en temiz yol.

developer ux: python sdk

plugin author decorator ile node yazar; sdk manifest üretir. elle devasa json yazmak zorunda kalmaz.

xml: adapter only

legacy/import/export veya belirli scientific metadata formatları için desteklenebilir; core spec olmamalı.

from synaptix import node, Input, Output

@node(
    id="eeg.mne.filter",
    inputs={"raw": Input("mne.Raw")},
    outputs={"filtered": Output("mne.Raw")},
)
def filter_raw(raw, l_freq: float, h_freq: float):
    return raw.filter(l_freq, h_freq)

docker security baseline

  --network none \
  --memory 16g \
  --cpus 4 \

  --security-opt no-new-privileges \

appendix

son karar log’u ve sunumda kullanılacak özet

türkçe pitch

synaptix, datasetleri, workflow graph’larını ve execution run’larını immutable snapshot olarak versiyonlayan; go control plane ile izinleri, marketplace entitlement’larını ve run state’i yöneten; python container’larıyla domain plugin’lerini güvenli şekilde çalıştırıp artifact, report ve provenance üreten low-code bilimsel workflow repository platformudur.

english pitch

synaptix is a low-code scientific workflow repository where datasets, workflow graphs and executions are versioned as immutable snapshots; a go control plane coordinates permissions, marketplace entitlements and run state, while isolated python containers execute domain plugins against s3-backed data and produce reproducible artifacts, reports and provenance.

nihai mimari kararlar

1. git-like product versioning, not raw git wrapping.
2. object storage for data, postgres for metadata/manifests/state.
3. json schema plugin contracts, python sdk for authors.
4. go control plane, python execution containers.
5. nats jetstream for job/event flow, postgres as source of truth.
6. docker runner first, k8s/argo backend later.
7. workflow draft mutable, workflow version immutable.
8. every run creates provenance, artifacts and report links.
9. marketplace uses entitlements and signed plugin packages.
10. local-first trust, hosted execution later as paid upgrade.
