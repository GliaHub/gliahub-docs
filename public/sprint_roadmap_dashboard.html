Gliahub Roadmap v2

Workflow-editor-first reproducible neuroscience platform roadmap

A complete 42-sprint company roadmap, with Sprints 1–8 scoped as MVP Core. The plan keeps the product centered on visual workflows, native versioning, local-first execution, anti-lock-in exports, and a staged path toward core facilities, marketplace, hosted execution, and enterprise readiness.

Sprints 1–8

MVP Core

Local execution, workflow editor, versioning, reports, BIDS validation.

Sprints 9–12

Pilot Release

Installer, large uploads, first lab migration, visual diff, public templates.

Sprints 13–18

Extensibility & Community

Plugin docs, Python SDK, MOABB/Braindecode, contributor path.

Sprints 19–24

Lab Product + Core Facility

Private workspaces, RBAC, audit, facility isolation, queue/usage reports.

Sprints 25–30

Marketplace & Scale

Plugin listings, entitlement gates, private plugins, NATS, GPU scheduling.

Sprints 31–42

Hosted / Enterprise

Security hardening, hosted control plane, Argo/K8s, SSO, archives, fMRI, notebooks.

Open-source boundary

Open specs, SDKs, CLI, local runner, basic EEG nodes, and export format. Commercial hosted control plane, billing, marketplace backend, enterprise admin/audit, and hosted execution.

Runner security evolution

Docker SDK for MVP, isolated runner hosts for pilot, containerd/gVisor/Firecracker for hosted/core facility, Kubernetes/Argo as a security and compliance backend as much as a scale backend.

Regulatory boundary

Research workflow and reproducibility platform only. No clinical diagnosis or medical decision support without a later regulatory product decision.

Sprint overview

Each sprint ends with a demonstrable product capability or validated learning outcome.
